## 二、前后端交互流程图

### 2.1 启动与路由分发（pages/home/index）

```
用户打开小程序
       │
       ▼
pages/home/index  onLoad
       │
       ├─── 调用第三方接口获取用户公网IP
       │    https://cz88.net/api/cz88/ip/openIPInfo
       │           │
       │           ▼
       │    [接口1] GET /biz/channel/getPageCodeByIp
       │    参数: { channelId, ip }
       │           │
       │           ├── 返回 "a"  ──────► navigateTo pages/other/index
       │           │
       │           └── 返回 URL  ──────► navigateTo pages/h5/index?url=<URL>
       │
       └─── IP获取失败 / 接口异常 ────────► navigateTo pages/other/index
```

---

### 2.2 主功能页（pages/other/index）onLoad 流程

```
进入 pages/other/index
       │
       ▼
[接口2] GET /infostream/rt2?channelId=xxx
返回: { UUID, ... }
       │
       ├─── [接口3] POST /biz/tEventRecordNew/addRecord
       │    参数:
       │      channelId       渠道ID
       │      eventType       "mini_page_open"
       │      uuid            rt2 返回的 UUID
       │      extendInfo (JSON字符串):
       │        pageId        "bg_alipay"
       │        timestamp     当前时间戳
       │        buttonKey     my.getEnterOptionsSync().scene（进入场景值）
       │        duration      0
       │        uuid          rt2 返回的 UUID
       │
       └─── [接口1] GET /biz/channel/getPageCodeByIp
            参数: { channelId }
                   │
                   ├── 返回 "a"  ──► 停留当前页，展示功能菜单
                   │
                   └── 返回 URL  ──► redirectTo pages/h5/index
                                     参数:
                                       url=<URL>
                                       alipayUuid=<UUID>
                                       zzUuId=<UUID>
                                       ...路由原始参数
```

**用户操作功能菜单：**

```
pages/other/index 功能菜单
       ├── 点击"汽车计算器" ──► navigateTo pages/buyCar/index
       ├── 点击"车主计算器" ──► navigateTo pages/carOwner/index
       └── 点击"用户服务协议" ──► navigateTo pages/link/index
```

---

### 2.3 购车/车主计算器（pages/buyCar/index & pages/carOwner/index）

```
进入计算器页面
       │
       onLoad ──► handlePageShow(option)
                   │  （与 2.2 完全相同的分流逻辑）
                   ├── [接口2] GET /infostream/rt2
                   ├── [接口3] POST /biz/tEventRecordNew/addRecord
                   └── [接口1] GET /biz/channel/getPageCodeByIp
                               ├── 返回 "a"  ──► 停留当前计算器页
                               └── 返回 URL  ──► redirectTo pages/h5/index
       │
用户输入参数，点击"开始计算"
       │
       └── 纯前端本地计算，不调用任何后端接口
           ├── 购车计算器：全款总费用（购置税/综合服务费/车船税等）、分期月供
           └── 车主计算器：等额本息 / 等额本金，输出月供、利息、还款总额
```

---

### 2.4 WebView页获取手机号（pages/h5/index）

```
pages/h5/index 加载 WebView
src = VITE_H5_BASEURL + url参数
       │
       H5页面触发获取手机号
       │
       H5 向小程序发送 postMessage:
       { type: "getPhoneNumber" }
       │
       ▼
小程序调用支付宝原生 API
my.getPhoneNumber()
       │
       ├── 成功获取加密response
       │           │
       │           ▼
       │   [接口4] POST /common/callback/alipayDecrypt
       │   参数: { resp: <支付宝返回的response>, channelId }
       │           │
       │           ├── errCode="0" 且 code="10000"
       │           │        └──► 获取 mobile（手机号明文）
       │           │             webViewContext.postMessage(mobile)
       │           │             回传给H5页面
       │           │
       │           └── 解密失败 / code 非10000
       │                    └──► navigateTo pages/Authorization/index
       │                         （用户重新授权，授权完成回调后重试）
       │
       └── 调用失败 ──► 打印错误日志，不处理
```

## 三、后端接口汇总

### 3.1 当前实际调用的接口

| # | 方法 | 接口路径 | 调用场景 | 请求参数 | 返回说明 |
|---|------|---------|---------|---------|---------|
| 1 | `GET` | `/biz/channel/getPageCodeByIp` | 启动分流 / 每个页面 onLoad | `channelId`、`ip`（可选） | `"a"` = 走原生页；其他值 = H5页URL |
| 2 | `GET` | `/infostream/rt2` | 每个功能页 onLoad | `channelId` | `{ UUID }` 用户唯一标识 |
| 3 | `POST` | `/biz/tEventRecordNew/addRecord` | 页面打开埋点上报 | `channelId`、`eventType`、`extendInfo`、`uuid` | — |
| 4 | `POST` | `/common/callback/alipayDecrypt` | 解密支付宝手机号 | `resp`（支付宝加密串）、`channelId` | `{ errCode, data: { code, mobile } }` |

## 四、接口请求公共规范

### 请求头

| 字段 | 说明 |
|------|------|
| `platform` | 当前运行平台标识（由 `src/utils/platform.ts` 提供） |
| `token` | 用户登录态 token（存在时自动附加） |

### 响应格式

```json
{
  "success": true,
  "errCode": "0",
  "data": "...",
  "message": "...",
  "msg": "..."
}
```

| 字段 | 说明 |
|------|------|
| `success` | `false` 时前端自动弹出 `message` 提示 |
| `errCode` | `"401"` 时清除登录态并跳转登录页 |

### 超时配置

- 全局请求超时：**100000ms（100秒）**
- manifest 配置超时：**180000ms（3分钟）**

## 六、关键设计说明

1. **分流机制**：`/biz/channel/getPageCodeByIp` 是整个小程序的路由枢纽。返回 `"a"` 走原生页面，返回任意 URL 则全部跳进 WebView 展示 H5，实现了原生与 H5 的动态切换。

2. **UUID 追踪链路**：`/infostream/rt2` 返回的 `UUID` 贯穿整个用户会话，作为埋点标识传入 `addRecord`，同时作为 `alipayUuid` / `zzUuId` 参数传递给 H5 页面。

3. **手机号获取链路**：H5无法直接调用支付宝 `getPhoneNumber`，需通过 `postMessage` 通知小程序，由小程序调用原生 API 获取加密串，再经后端解密后回传明文给 H5。

4. **计算器纯前端实现**：购车计算器（全款/分期）和车主计算器（等额本息/等额本金）的所有计算逻辑全部在前端本地完成，不依赖任何后端接口。

5. **待接入接口**：`alipayCallBack`、`alipayPageViewCallBack` 在代码中已有定义和引入，但相关调用均已注释，可能是规划中待接入或已废弃的功能点。
