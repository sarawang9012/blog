## 三、各接口详细规范

### 3.1 GET `/biz/channel/getPageCodeByIp` — A/B 面分流

**调用场景**：小程序启动时、每个页面 onLoad 时，判断当前用户应该看到 A 面（小程序原生）还是 B 面（WebView H5）。

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| channelId | String | 是 | 渠道 ID |
| ip | String | 否 | 用户公网 IP（前端可选传入，不传时后端从请求头获取） |

**请求示例**：
```
GET /biz/channel/getPageCodeByIp?channelId=301&ip=120.xxx.xxx.xxx
```

**响应格式**：
```json
{
  "success": true,
  "errCode": "0",
  "data": "a"
}
```

- `data = "a"` → A 面（小程序原生页面）
- `data = "b"` → B 面（跳转 WebView 加载 H5 落地页）
- 异常时默认返回 `"a"`

**核心业务逻辑**（来自原始项目 `ChannelPageServiceImpl.getCityInfoByIp`）：

1. **IP 地理位置查询**：使用纯真 IP 数据库（cz88）查询 IP 对应的省市
   - 依赖：`net.cz88.czdb.DbSearcher`，`QueryType.MEMORY` 模式
   - 配置项：
     ```yaml
     chunzhen:
       ip:
         path: /path/to/cz88_public_v4.czdb
         secret: <密钥>
     ```
   - 查询结果格式：`"Country\u2013Province\u2013City"`，以 `\u2013`（en-dash）分隔

2. **省市解析**：从查询结果中提取省份（index 1）和城市（index 2）

3. **A/B 面路由决策**：查询 `channel_page` 表，根据 `channelId` 获取配置规则
   - **优先级 1 — 排除城市**：如果 `mismatchCityName` 包含该省或市 → 返回 `mismatchPageCode`
   - **优先级 2 — 匹配省份**：如果 `matchProvinceName` 包含该省 → 返回 `matchPageCode`
   - **默认**：返回 `defaultPageCode`

4. **异常兜底**：任何错误返回 `"a"`

**IP 获取逻辑**（来自原始项目 `ChannelController.getClientIpAddress`）：

按以下顺序从请求头获取客户端真实 IP：
1. `X-Forwarded-For`（取逗号前的第一个 IP）
2. `X-Real-IP`
3. `Proxy-Client-IP`
4. `WL-Proxy-Client-IP`
5. `HTTP_X_FORWARDED_FOR`
6. `HTTP_X_REAL_IP`
7. `HTTP_PROXY_CLIENT_IP`
8. `HTTP_WL_PROXY_CLIENT_IP`
9. 兜底：`request.getRemoteAddr()`

**数据库表 `channel_page`**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String (INPUT) | 主键 |
| channelId | String | 渠道 ID |
| matchProvinceCode | String | 匹配省份编码 |
| matchProvinceName | String | 匹配省份名称（逗号分隔，contains 判断） |
| mismatchCityCode | String | 排除城市编码 |
| mismatchCityName | String | 排除城市名称（逗号分隔，contains 判断） |
| defaultPageCode | String | 默认页面编码（兜底） |
| matchPageCode | String | 匹配时的页面编码 |
| mismatchPageCode | String | 排除时的页面编码 |
| createTime | LocalDateTime | 创建时间 |
| operTime | LocalDateTime | 修改时间 |
| deleteFlag | Integer | 0=有效，1=已删除 |

**原始项目参考文件**：
- Controller：`zxw-loan-market-api/.../infostream/controller/ChannelController.java`（line 122）
- Service 接口：`zxw-loan-market-common/.../biz/service/ChannelPageService.java`
- Service 实现：`zxw-loan-market-common/.../biz/service/impl/ChannelPageServiceImpl.java`（line 124）
- Entity：`zxw-loan-market-common/.../biz/entity/ChannelPageEntity.java`
- DAO：`zxw-loan-market-common/.../biz/dao/ChannelPageDao.java`
- Mapper XML：`zxw-loan-market-common/.../resources/mapper/biz/ChannelPageDao.xml`

- `loan-market-common`：`ChannelPageService` + `ChannelPageServiceImpl` + `ChannelPageEntity` + `ChannelPageDao` **已存在**
- `loan-market-api`：**无** Controller 端点
- `loan-market admin`：`ChannelController`（映射 `biz/channel`）**无** `getPageCodeByIp` 端点

**需实现**：在 `loan-market-api` 的 `ChannelController` 中新增 `getPageCodeByIp` 端点

### 3.2 GET `/infostream/rt2` — UUID 生成与渠道配置

**调用场景**：每个功能页 onLoad 时调用，获取用户追踪 UUID，贯穿整个会话的埋点链路。

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| channelId | String | 是 | 渠道 ID |

**请求示例**：
```
GET /infostream/rt2?channelId=301
```

**响应格式**（简化版，当前前端只用 UUID）：
```json
{
  "success": true,
  "errCode": "0",
  "data": {
    "UUID": "generated-uuid-string"
  }
}
```

**核心业务逻辑**（来自原始项目 `InfoStreamController.rt2`）：

1. 生成 UUID（`Utils.getUUID()`）
2. 按 channelId 查询 `url_mappings` 表获取渠道 URL 映射
3. 如有 `pageConfigId`，查询 `infostream_page_config` 获取页面配置
4. 如有页面配置，查询 `page_element_config` 获取页面元素配置
5. 组装返回数据

**当前项目需求**：简化实现，返回 UUID 即可。后续按需扩展页面配置查询。

**原始项目完整返回结构**：
```json
{
  "channelId": "301",
  "UUID": "generated-uuid-string",
  "pageCode": "a_or_b",
  "pageElementConfig": { ... },
  "typeCode": "some_type",
  "label": "some_label"
}
```

**原始项目参考文件**：
- Controller：`zxw-loan-market-api/.../infostream/controller/InfoStreamController.java`（line 168）
- UrlMappingEntity：`zxw-loan-market-common/.../biz/entity/UrlMappingEntity.java`（表 `url_mappings`）
- InfostreamPageConfigEntity：`zxw-loan-market-common/.../biz/entity/InfostreamPageConfigEntity.java`（表 `infostream_page_config`）
- PageElementConfigEntity：`zxw-loan-market-common/.../biz/entity/PageElementConfigEntity.java`（表 `page_element_config`）

**现有代码状态**：
- `loan-market-api`：仅有 `/infostream/rt`（HTTP 302 重定向版），**无** `/rt2`
- `loan-market-common`：`UrlMappingEntity` / `UrlMappingService` 已存在

**需实现**：在 `loan-market-api` 的 `InfoStreamController` 中新增 `/rt2` 端点

### 3.3 POST `/biz/tEventRecordNew/addRecord` — 埋点上报

**调用场景**：页面打开、用户操作时上报事件埋点，关联 UUID 追踪链路。

**请求参数**（`TEventRecordNewForm`，JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| channelId | Integer | 否 | 渠道 ID |
| eventType | String | 是 | 事件类型（如 `"mini_page_open"`） |
| uuid | String | 否 | rt2 返回的 UUID |
| pid | String | 否 | 单个产品 ID |
| pids | String | 否 | 多个产品 ID（逗号分隔），默认 `""` |
| phone | String | 否 | 手机号 |
| phoneMd5 | String | 否 | 手机号 MD5 |
| invite | String | 否 | 渠道邀请码 |
| extendInfo | String | 否 | JSON 字符串，扩展信息（见下方） |
| ip | String | 否 | 客户端 IP |

**extendInfo JSON 结构**：
```json
{
  "pageId": "bg_alipay",
  "timestamp": 1713782400000,
  "duration": 0,
  "buttonKey": "scene_value",
  "label": "",
  "clueId": "",
  "cid": "",
  "productName": "",
  "platform": "",
  "adId": ""
}
```

**前端调用示例**（参考项目 `src/utils/index.ts`）：
```ts
await addRecordFn(channelId, rt2?.data || {})
// 参数:
// channelId, eventType="mini_page_open", uuid=rt2.UUID,
// extendInfo = JSON.stringify({
//   pageId: "bg_alipay",
//   timestamp: Date.now(),
//   buttonKey: my.getEnterOptionsSync().scene,
//   duration: 0,
//   uuid: rt2.UUID
// })
```

**响应格式**：
```json
{
  "code": 0,
  "msg": "success"
}
```
（注：该接口在原始项目中使用 `R` 返回类型而非 `APIResult`，`code=0` 表示成功）

**核心业务逻辑**（来自原始项目 `TEventRecordNewController.addRecord`）：

1. **获取客户端 IP**：`ServletUtil.getClientIP(request)`

2. **用户识别**（两种策略）：
   - 优先：从 `Authorization` 请求头解析 JWT token，提取 userId，加载 `UserEntity`
   - 备选：通过 `phoneMd5` 查找用户
   - 都无法识别时返回错误 "用户不存在！"

3. **事件类型校验**：按 `eventType` 查询 `t_event` 表，不存在则返回 "没有该埋点事件！"

4. **渠道解析**（两种策略）：
   - 优先：按 `channelId` 查询 `ChannelEntity`
   - 备选：按 `invite` 邀请码查询
   - 都无法解析时 cid 设为 0

5. **创建记录**：
   - 单条：`BeanUtils.copyProperties()` → `tEventRecordNewService.save()`
   - 多条（pids 不为空且不含逗号时）：拆分 pids → `saveBatch()`

6. **记录字段填充**（`getEventRecordNew()` 辅助方法）：
   - `uid` ← 用户 ID
   - `cid` ← 渠道 ID（或用户 channelId，或 0）
   - `eventId` ← t_event 表查询的事件 ID
   - `ip` ← `IPUtils.getIpAddr(request)`
   - `createTime` ← `new Date()`
   - `pid` ← 产品 ID

7. **扩展信息解析**（`extendInfo()` 方法）：解析 extendInfo JSON，提取 pageId、timestamp、duration、buttonKey、label、clueId、cid(customerId)、productName、platform、adId 等

**数据库表 `t_event_record_new`**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer (AUTO) | 自增主键 |
| cid | Integer | 渠道 ID |
| uid | Integer | 用户 ID |
| eventId | Integer | 事件类型 ID（FK → t_event） |
| uuid | String | 追踪 UUID |
| ip | String | 客户端 IP |
| pid | String | 产品 ID |
| createTime | Date | 创建时间 |
| pageId | String | 页面标识 |
| duration | Long | 停留时间(ms) |
| timestamp | Date | 点击时间 |
| buttonKey | String | 按钮标识 |
| label | String | 协议标签 |
| clueId | String | 线索 ID |
| customerId | String | 客户 ID |
| productName | String | 下游产品名称 |
| platform | String | 平台名称 |
| adid | String | 支付宝广告创意 ID |

**数据库表 `t_event`**（事件类型字典）：

已知事件 ID 映射：
- 151 = UV（页面浏览）
- 152 = 注册
- 154 = H5 留资提交
- 155 = H5 贷超点击
- 156 = H5 弹窗点击
- 159 = 下载

**原始项目参考文件**：
- Controller：`zxw-loan-market-api/.../infostream/controller/TEventRecordNewController.java`（line 65）
- Service 接口：`zxw-loan-market-common/.../biz/service/TEventRecordNewService.java`
- Service 实现：`zxw-loan-market-common/.../biz/service/impl/TEventRecordNewServiceImpl.java`
- Form DTO：`zxw-loan-market-common/.../biz/form/TEventRecordNewForm.java`

- DAO：`zxw-loan-market-common/.../biz/dao/TEventRecordNewDao.java`

### 3.4 POST `/common/callback/alipayDecrypt` — 支付宝手机号解密

**调用场景**：H5 页面通过 postMessage 请求小程序获取手机号 → 小程序调用 `my.getPhoneNumber` → 获取加密串 → 后端解密 → 回传明文给 H5。

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| resp | String | 是 | 支付宝 `my.getPhoneNumber` 返回的加密 response |
| channelId | String | 否 | 渠道 ID |

**请求示例**：
```json
{
  "resp": "encrypt_response_string_from_alipay",
  "channelId": "301"
}
```

**响应格式**：
```json
{
  "errCode": "0",
  "success": true,
  "data": "{\"code\":\"10000\",\"mobile\":\"13800138000\"}"
}
```

- `errCode = "0"` 且 `data` 中 `code = "10000"` → 解密成功，`mobile` 为手机号明文
- 解密失败 → 前端跳转授权页面重新授权

**核心业务逻辑**：

1. 接收 `AlipayDecryptDTO`（含 `resp` 和 `channelId`）
2. 委托 `callBackService.alipayDecrypt()` 执行解密
3. 解密使用渠道对应的 AES 密钥

**现有代码状态**：
- `loan-market-api`：`CallBackController.alipayDecrypt()` **已存在**，完整实现
  - 文件：`loan-market-api/.../open/controller/CallBackController.java`（line 163）
  - 类映射：`@RequestMapping("common/callback/")`
  - 方法映射：`@PostMapping("/alipayDecrypt")`

**需实现**：**无需新增**，确认支付宝 AES 密钥配置正确即可
