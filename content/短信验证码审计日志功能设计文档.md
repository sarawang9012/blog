# 短信验证码审计日志功能设计文档

## 1. 功能概述

短信验证码审计日志功能用于记录所有短信验证码发送请求的详细信息，包括：
- 请求来源（渠道ID、渠道名称、IP地址）
- 手机号（DES加密存储、脱敏展示、MD5哈希）
- 请求结果（成功、重复发送、频率限制、渠道错误、参数错误、手机号错误、系统异常）
- 请求耗时

该功能采用 **异步写入** 方式，不影响主业务流程性能。

---

## 2. 技术选型：AOP 切面编程

### 2.1 什么是 AOP？

AOP（Aspect-Oriented Programming，面向切面编程）是一种编程范式，用于将 **横切关注点**（如日志、事务、安全）与核心业务逻辑分离。

**核心概念：**

| 概念             | 说明          | 示例                            |
| -------------- | ----------- | ----------------------------- |
| 切面（Aspect）     | 横切逻辑的模块     | `SmsCaptchaAuditAspect`       |
| 连接点（JoinPoint） | 程序执行过程中的某个点 | 方法调用                          |
| 切入点（Pointcut）  | 匹配连接点的表达式   | `execution(* ...smsSend(..))` |
| 通知（Advice）     | 在切入点执行的代码   | `@Around` 环绕通知                |

### 2.2 为什么选择 AOP？

- **非侵入式**：不需要修改原有业务代码
- **统一管理**：所有审计逻辑集中在一个切面类中
- **易于维护**：新增或修改审计逻辑只需修改切面类
- **自动生效**：匹配切入点的方法自动被拦截

## 4. 关键实现逻辑

### 4.1 切入点定义（Pointcut）

```java
// 拦截 H5 渠道短信发送方法
@Pointcut("execution(* io.renren.modules.open.service.impl.CommonApiServiceImpl.smsSend(..))")
public void smsSendPointcut() {}

// 拦截 APP 端短信发送方法
@Pointcut("execution(* io.renren.modules.app.controller.AppSmsController.send(..))")
public void appSmsSendPointcut() {}

// 拦截 APP 加密端短信发送方法
@Pointcut("execution(* io.renren.modules.app.controller.encr.AppSmsEncrController.send(..))")
public void appSmsEncrSendPointcut() {}
```

**切入点表达式说明：**
- `execution(...)` 表示匹配方法执行
- 第一个 `*` 表示任意返回类型
- `(..)` 表示任意参数列表

### 4.2 环绕通知（@Around）

```java
@Around("smsSendPointcut()")
public Object aroundSmsSend(ProceedingJoinPoint joinPoint) throws Throwable {
    long startTime = System.currentTimeMillis();  // 记录开始时间
    Object result = null;
    Throwable throwable = null;

    try {
        result = joinPoint.proceed();  // 执行原方法
        return result;
    } catch (Throwable e) {
        throwable = e;
        throw e;  // 异常继续抛出，不影响原逻辑
    } finally {
        try {
            long elapsed = System.currentTimeMillis() - startTime;
            buildAndSaveLogForCommonApi(joinPoint, result, throwable, elapsed);
        } catch (Exception ex) {
            log.error("短信验证码审计日志切面处理异常, error={}", ex.getMessage(), ex);
        }
    }
}
```

**执行流程：**
1. 记录请求开始时间
2. 执行原业务方法（`joinPoint.proceed()`）
3. 捕获异常（如有）并重新抛出
4. 在 `finally` 块中构建并保存审计日志
5. 即使审计逻辑失败，也不影响主业务

### 4.3 参数提取

不同接口使用不同的 DTO 类型：

| 接口 | DTO 类型 | 手机号获取方式 |
|------|----------|---------------|
| CommonApiServiceImpl.smsSend | `CommonApiDto` | 解密 `data` 字段获取 `phone` |
| AppSmsController.send | `SendSmsDto` | 直接获取 `mobile` 字段 |
| AppSmsEncrController.send | `SendSmsDto` | 直接获取 `mobile` 字段 |

**H5 渠道手机号提取逻辑：**
```java
private String extractPhone(CommonApiDto dto, SmsCaptchaAuditLogEntity entity) {
    // 1. 获取渠道ID
    Integer channelId = dto.getChannelId();
    
    // 2. 查询渠道信息
    ChannelEntity channelEntity = channelService.getById(channelId);
    entity.setChannelName(channelEntity.getName());
    
    // 3. 使用渠道的 AES 密钥解密 data 字段
    String decrypt = JavaAesCommonUtils.aesDecrypt(dto.getData(), channelEntity.getAesKey());
    
    // 4. 从 JSON 中提取 phone
    JSONObject data = JSONUtil.toBean(decrypt, JSONObject.class);
    return data.getStr("phone");
}
```

**APP 端手机号提取逻辑：**
```java
String mobile = sendSmsDto.getMobile();

// 如果不是标准手机号格式（可能是 DES 加密的），则解密
if (mobile != null && !mobile.matches("^1[3-9]\\d{9}$")) {
    mobile = jsDesUtils.decode(mobile);
}
```

### 4.4 手机号安全处理

```java
// DES 加密存储（用于安全存储）
entity.setMobileCipher(jsDesUtils.encode(phone));

// 脱敏展示（用于前端展示）
entity.setMobileMask(maskMobile(phone));  // 138****1234

// MD5 哈希（用于查询和分组统计）
entity.setMobileMd5(MD5Util.encrypt32(phone));
```

### 4.5 请求结果判定

| requestResult | 含义 | 判定条件 |
|---------------|------|----------|
| 0 | 初始化/未知 | 默认值 |
| 1 | 发送成功 | `code == 0` 或消息包含"验证码已下发" |
| 2 | 重复发送拦截 | 消息包含"连续输错"或"黑名单" |
| 3 | 频率限制拦截 | 消息包含"滑动验证"或"ip限制" |
| 4 | 渠道无效 | `code == ERROR_CHANNEL` |
| 5 | 参数错误 | `code == ERROR_PARAM` |
| 6 | 手机号格式错误 | `code == ERROR_PHONE` |
| 7 | 系统异常 | 捕获到异常或 `code == ERROR_SYSTEM` |

### 4.6 异步写入机制

```java
// SmsCaptchaAuditLogService.java
public void saveAuditLogAsync(SmsCaptchaAuditLogEntity entity) {
    // 使用独立线程池异步执行
    executorService.submit(() -> {
        try {
            smsCaptchaAuditLogDao.insert(entity);
        } catch (Exception e) {
            log.error("审计日志异步保存失败, entity={}", entity, e);
        }
    });
}
```

**优势：**
- 不阻塞主业务流程
- 即使数据库写入失败，也不影响短信发送
- 使用独立线程池，避免影响其他异步任务

## 5. 数据库表结构

```sql
CREATE TABLE `sms_captcha_audit_log` (
    `id` varchar(36) NOT NULL COMMENT '主键ID',
    `request_no` varchar(64) DEFAULT NULL COMMENT '请求流水号',
    `channel_id` varchar(36) DEFAULT NULL COMMENT '渠道ID',
    `channel_name` varchar(100) DEFAULT NULL COMMENT '渠道名称',
    `mobile_cipher` varchar(255) DEFAULT NULL COMMENT '手机号DES加密存储',
    `mobile_mask` varchar(20) DEFAULT NULL COMMENT '手机号脱敏展示',
    `mobile_md5` varchar(32) DEFAULT NULL COMMENT '手机号MD5哈希',
    `ip_address` varchar(45) DEFAULT NULL COMMENT '请求IP',
    `sms_type` varchar(50) DEFAULT NULL COMMENT '短信业务类型',
    `request_result` int NOT NULL DEFAULT '0' COMMENT '请求结果: 0-初始化 1-成功 2-重复拦截 3-频率限制 4-渠道无效 5-参数错误 6-手机号错误 7-系统异常',
    `response_msg` varchar(255) DEFAULT NULL COMMENT '响应消息',
    `elapsed_time` int DEFAULT NULL COMMENT '请求耗时(毫秒)',
    `error_msg` text DEFAULT NULL COMMENT '异常信息',
    `create_user_id` varchar(36) DEFAULT NULL,
    `create_time` datetime(6) DEFAULT NULL,
    `oper_user_id` varchar(36) DEFAULT NULL,
    `oper_time` datetime(6) DEFAULT NULL,
    `status` int NOT NULL DEFAULT '1' COMMENT '0-初始化 1-正常 3-逻辑删除',
    PRIMARY KEY (`id`),
    KEY `idx_mobile_md5` (`mobile_md5`),
    KEY `idx_channel_id` (`channel_id`),
    KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 8. 注意事项

1. **切面异常不影响主业务**：所有审计逻辑在 `try-catch` 中执行，即使失败也只记录日志
2. **手机号必须加密存储**：使用 `jsDesUtils.encode()` 加密，不允许明文存储
3. **异步写入可能延迟**：日志写入是异步的，高并发时可能有短暂延迟
4. **线程池配置**：异步线程池在 `ThreadPoolConfig` 中配置，名称前缀为 `sms-audit`
